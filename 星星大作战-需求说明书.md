
# 星星大作战 - 一年级学生习惯养成计划系统需求说明书

## 1. 项目概述

### 1.1 项目背景
为帮助一年级学生养成良好的日常行为习惯，通过游戏化的方式（星星奖励、勋章体系、礼品兑换）激发学生的积极性，设计这款童趣多彩的习惯养成管理系统。

### 1.2 项目目标
- 让习惯养成变得有趣、有动力
- 提供直观的任务管理功能
- 通过奖励机制激励学生持续坚持
- 培养学生的责任感和成就感

### 1.3 目标用户
- 主要用户：一年级女生（6-7岁）
- 辅助用户：爸爸妈妈（家长）

---

## 2. 功能需求

### 2.1 任务管理模块

#### 2.1.1 日常任务
- **功能描述**：管理学生的日常任务
- **任务属性**：
  - 任务名称
  - 任务类型（日常任务/行为习惯）
  - 奖励星星数
  - 是否为连续性任务
  - 连续天数要求（如为连续性任务）

#### 2.1.2 行为习惯
- **功能描述**：管理学生的行为习惯类任务
- **与日常任务的区别**：侧重于长期行为培养，如"礼貌用语"、"整理书包"等

#### 2.1.3 任务操作
- **新增任务**：
  - 输入任务名称
  - 选择任务类型
  - 设置奖励星星数
  - 设置是否为连续性任务
  - 如为连续性任务，设置连续天数要求

- **编辑任务**：
  - 修改任务名称
  - 修改奖励星星数
  - 修改连续性设置

- **删除任务**：
  - 删除不再需要的任务

- **完成任务**：
  - 标记任务为已完成
  - 自动获得相应星星数
  - 连续性任务自动记录连续天数

### 2.2 星星兑换模块

#### 2.2.1 兑换表管理
- **功能描述**：管理可兑换的礼品
- **礼品属性**：
  - 礼品名称
  - 所需星星数
  - 礼品描述
  - 礼品图片（可选）

#### 2.2.2 礼品展示
- **功能描述**：在页面展示可兑换的礼品
- **展示要求**：
  - 按所需星星数从低到高排序
  - 高亮显示当前星星数可兑换的礼品
  - 显示距离下一个可兑换礼品还需多少星星

#### 2.2.3 兑换操作
- **功能描述**：兑换礼品
- **操作流程**：
  - 选择要兑换的礼品
  - 确认兑换
  - 自动扣除相应星星数
  - 记录兑换历史

### 2.3 勋章体系模块

#### 2.3.1 勋章设计
- **勋章等级**：
  - 青铜勋章：累计10颗星星
  - 白银勋章：累计30颗星星
  - 黄金勋章：累计60颗星星
  - 钻石勋章：累计100颗星星
  - 皇冠勋章：累计200颗星星

- **神秘奖励**：
  - 集齐青铜、白银、黄金勋章 → 解锁"星星小勇士"称号
  - 集齐所有勋章 → 解锁"星星大作战王者"神秘大奖

#### 2.3.2 勋章展示
- **功能描述**：展示学生已获得的勋章和进度
- **展示要求**：
  - 已获得的勋章点亮显示
  - 未获得的勋章灰色显示
  - 显示距离下一个勋章还需多少星星
  - 展示神秘奖励解锁进度

### 2.4 星星统计模块

#### 2.4.1 星星概览
- **功能描述**：展示星星获取情况
- **展示内容**：
  - 当前总星星数
  - 今日获得星星数
  - 本周获得星星数
  - 累计获得星星数

#### 2.4.2 任务完成统计
- **功能描述**：展示任务完成情况
- **展示内容**：
  - 今日完成任务数
  - 连续完成天数
  - 各类型任务完成率

### 2.5 家长管理模块

#### 2.5.1 家长权限
- **功能描述**：爸爸妈妈可以管理孩子的任务和奖励
- **权限功能**：
  - 添加、编辑、删除任务
  - 设置奖励规则
  - 管理礼品兑换表
  - 查看孩子的详细统计数据

#### 2.5.2 家长视图
- **功能描述**：为家长提供专属的管理界面
- **视图内容**：
  - 孩子的任务完成情况汇总
  - 星星获取趋势图表
  - 勋章获得进度
  - 兑换历史记录
  - 习惯养成建议

## 3. 界面设计要求

### 3.1 整体风格
- **主题**：星星大作战（粉色公主风）
- **色彩**：童趣多彩，以粉色为主的甜美配色方案
  - 主色调：粉色（#FFB6C1）、浅粉色（#FFC0CB）、玫瑰粉（#FF69B4）
  - 辅助色：金色（#FFD700）、白色、淡紫色（#E6E6FA）、浅蓝色（#ADD8E6）
- **风格**：卡通化、游戏化、公主风
- **元素**：星星、彩虹、云朵、花朵、蝴蝶、小公主、皇冠等童趣元素

### 3.2 页面布局

#### 3.2.1 首页（星星 dashboard）
- **顶部**：当前总星星数（大号字体，星星图标）
- **中部**：
  - 勋章展示区（横向排列）
  - 今日任务快速完成区
- **底部**：导航栏（任务、兑换、我的）

#### 3.2.2 任务页面
- **顶部**：任务分类标签（日常任务/行为习惯）
- **中部**：任务列表
  - 每个任务卡片包含：任务名称、星星数、完成按钮、连续天数（如适用）
- **浮动按钮**：新增任务按钮（+号）

#### 3.2.3 兑换页面
- **顶部**：当前星星数
- **中部**：礼品展示墙
  - 每个礼品卡片包含：礼品图片、名称、所需星星数、兑换按钮
  - 可兑换的礼品高亮显示
- **下部**：兑换历史记录

#### 3.2.4 我的页面
- **顶部**：学生头像、昵称
- **中部**：
  - 星星统计图表
  - 勋章墙（完整展示）
  - 神秘奖励解锁进度
- **下部**：设置选项

### 3.3 交互设计
- **动画效果**：
  - 完成任务时星星跳动动画
  - 获得勋章时庆祝动画
  - 兑换礼品时礼花效果
- **反馈**：
  - 操作成功/失败的提示
  - 星星数变化的动画反馈
- **操作**：
  - 简洁直观，适合一年级学生操作
  - 大按钮，易于点击

## 4. 数据需求

### 4.1 数据实体

#### 4.1.1 任务（Task）
```
{
  id: 任务ID
  name: 任务名称
  type: 任务类型（daily/habit）
  stars: 奖励星星数
  isContinuous: 是否为连续性任务
  continuousDays: 连续天数要求
  createdAt: 创建时间
}
```

#### 4.1.2 任务完成记录（TaskCompletion）
```
{
  id: 记录ID
  taskId: 任务ID
  completedAt: 完成时间
  starsEarned: 获得星星数
  continuousDay: 连续天数（如适用）
}
```

#### 4.1.3 礼品（Gift）
```
{
  id: 礼品ID
  name: 礼品名称
  starsRequired: 所需星星数
  description: 礼品描述
  image: 礼品图片URL
}
```

#### 4.1.4 兑换记录（Redemption）
```
{
  id: 记录ID
  giftId: 礼品ID
  redeemedAt: 兑换时间
  starsUsed: 使用星星数
}
```

#### 4.1.5 勋章（Medal）
```
{
  id: 勋章ID
  name: 勋章名称
  level: 勋章等级
  starsRequired: 所需星星数
  icon: 勋章图标
  isUnlocked: 是否已解锁
  unlockedAt: 解锁时间
}
```

#### 4.1.6 用户（User）
```
{
  id: 用户ID
  name: 学生姓名
  avatar: 头像
  currentStars: 当前星星数
  totalStarsEarned: 累计获得星星数
  consecutiveDays: 连续完成天数
}
```

## 5. 技术需求

### 5.1 技术栈建议
- **前端**：React + TypeScript + Tailwind CSS + shadcn/ui
- **后端**：Node.js + Express 或 Next.js API Routes
- **数据库**：SQLite（轻量级）或 PostgreSQL
- **存储**：LocalStorage（本地存储）或云存储

### 5.2 性能要求
- 页面加载时间 &lt; 2秒
- 操作响应时间 &lt; 500ms
- 支持离线使用（本地存储）

### 5.3 兼容性要求
- 支持主流浏览器（Chrome、Safari、Firefox、Edge）
- 支持移动端（手机、平板）
- 响应式设计
